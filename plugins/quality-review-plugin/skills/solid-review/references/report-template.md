## Notes for the report author (Claude)

- **Problematic code sections** must be removed entirely (not left blank) when status is ✅.
- Quotes should be **short excerpts**, not entire files. 5–25 lines is ideal.
- The **recommendation** should name concrete classes, interfaces, or patterns — not just say
  "apply the principle".
- If a principle is **not applicable** (e.g., procedural code has no interfaces), assign 0.5
  and explain why in the "What was found" section.
- The **Final verdict** should be the most human part of the report — motivating, honest, and
  pointing at the single highest-leverage improvement.
- Write in **the same language the user is using** (Spanish, English, Portuguese, etc.).
